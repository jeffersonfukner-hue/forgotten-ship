# O que é uma Sprint

A Sprint é a memória técnica oficial do projeto.

Ela não é apenas um changelog.

Ela é simultaneamente:

- documentação técnica;
- diário de desenvolvimento;
- material didático;
- roteiro para videoaulas;
- capítulo do futuro ebook;
- referência oficial para retomada do desenvolvimento.

Todo o conteúdo deve ser escrito pensando em ensinar outro desenvolvedor e, ao mesmo tempo, permitir que o projeto seja retomado meses depois sem depender do histórico da conversa.

---

# Regras da Sprint

## Documento Vivo

A Sprint é um documento vivo.

Ela deve ser atualizada continuamente durante o desenvolvimento.

Nunca deve ser deixada para ser escrita apenas ao final.

---

## Momento da Atualização

A Sprint deve ser atualizada sempre que houver quantidade suficiente de informação para justificar um novo registro.

Isso pode acontecer, por exemplo:

- após algumas missões;
- após uma refatoração importante;
- após uma decisão arquitetural relevante;
- após a implementação de um sistema completo;
- antes que uma grande quantidade de contexto seja perdida.

Não existe um número fixo de missões.

O critério é preservar conhecimento.

---

## Objetivo Principal

A Sprint deve conter informação suficiente para que qualquer desenvolvedor consiga compreender:

- o estado atual do projeto;
- as decisões tomadas;
- os motivos dessas decisões;
- os erros encontrados;
- o caminho seguido durante o desenvolvimento.

Sem necessidade de consultar o histórico da conversa.

---

## Consulta Obrigatória

Sempre que uma nova sessão de desenvolvimento começar, a Sprint deverá ser consultada antes da implementação de novas funcionalidades.

# Critério de Qualidade

Uma Sprint somente é considerada concluída quando ela permite:

✅ Retomar o projeto sem consultar o histórico da conversa.

✅ Explicar todas as decisões arquiteturais tomadas.

✅ Servir como roteiro para uma videoaula.

✅ Servir como capítulo do ebook.

✅ Registrar os erros e as soluções encontradas.

✅ Preservar o conhecimento produzido durante a Sprint.
